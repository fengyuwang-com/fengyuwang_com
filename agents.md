# Blog 项目结构与内容创作指南

## 项目概览

两个目录，互为源码与产物：

```
Desktop/
├── hugo-site/                 ← Hugo 源码项目
│   ├── hugo.toml              ← 配置（三语、每页24篇、RSS）
│   ├── content/zh-cn/blog/    ← 中文博客文章（Markdown）
│   │   ├── _index.md          ← 博客列表页标题/描述
│   │   └── posts/{slug}/index.md  ← 每篇文章一个文件夹
│   ├── layouts/blog/
│   │   ├── list.html          ← 卡片列表页模板（搜索+分页）
│   │   └── single.html        ← 文章页模板
│   ├── static/                ← 静态资源
│   └── deploy-blog.ps1        ← 一键构建+部署脚本
│
└── fengyuwang_5-deploy/       ← 线上部署目录
    ├── zh-cn/blog/             ← 中文博客（构建产物）
    ├── zh-hk/blog/             ← 繁体博客
    └── en/blog/                ← 英文博客
```

## 当前文章结构（每篇文章的 index.md）

```yaml
---
title: "文章标题"
description: "全文概要——显示在卡片列表上，也是正文第一段"
slug: "url-slug"
draft: false
---

正文第一段（与 description 相同，作为自然的文章引入）

正文第二段开始...
正文内容...
```

### description 的作用

- **卡片列表页**：显示在卡片上，作为摘要，帮助读者判断是否点开
- **文章页**：作为正文第一段，不加任何标签，自然引入

两者是同一段文字，在两个场景下有不同的阅读体验，不冲突。

## 模板逻辑

### 卡片列表页（list.html）

- 响应式网格布局：桌面多列，手机单列
- 每张卡片显示：标题 + 分隔线 + 概要文字（description）+ 分类标签
- 整张卡片可点击
- 顶部搜索框，实时过滤标题/概要/标签
- 底部居中分页：← 上一页 + 页码按钮 + 下一页 → + GO跳转
- 每页 24 篇文章

### 文章页（single.html）

- 顶部「← 返回博客」圆角按钮
- 标题
- 正文（第一段 = description，自然引入）
- 无日期、无摘要标签

## 文章改写指南

### 核心原则

description 和正文不能脱节。正文第一段用 description 的文字自然引入，后续内容与第一段逻辑衔接。

### 改写步骤

1. **读全文**：理解文章的核心观点和情绪基调
2. **写 description**：概括全文——这段文字会出现在卡片上，决定读者是否点进来。它也是正文第一段，所以要写得既像概要又像自然的文章开头
3. **改写正文**：
   - 保留原文的所有核心观点、案例、逻辑链条
   - 调整语气和表达方式
   - 确保第一段（description）自然衔接第二段

### 风格要求

**松弛感、真诚叙事**——像随手发的朋友圈或小红书笔记，真实有细节。

**文风：**
- 冷静而戏谑，俏皮而深刻
- 短句为主，口语化但有质感
- 可以带小吐槽 + 小感悟（基于内容，不硬凹）
- 留白感强，不说满

**结构：**
- 标题：带网感、有情绪或场景感
- 正文：真实体验 + 内心独白 + 细节场景
- 标签：3-5个，精准贴合内容 + 流量词

**禁忌：**
- ❌ "谁懂啊家人们" 等烂大街模板
- ❌ 刻意煽情、喊口号
- ❌ 逻辑严密但空洞
- ❌ 第二人称对话 ("我给你" "你如何如何")

**要求：**
- 不要求逻辑严密，真实就好
- 突出个人感受和生活洞察

## 工作流

### 写新文章

1. 在 `content/zh-cn/blog/posts/{slug}/` 下创建 `index.md`
2. 写 frontmatter：title、description、slug、draft
3. 正文第一段与 description 相同
4. 后续正文与第一段逻辑衔接
5. 运行 `deploy-blog.ps1` 构建并部署

### 翻译

复制 `zh-cn/blog/posts/{slug}/` 到 `zh-hk/` 和 `en/`，翻译 frontmatter 和正文内容。
