# Blog 项目结构与内容创作指南

## 项目概览

两个目录，互为源码与产物：

```
Desktop/
├── hugo-site/                   ← Hugo 源码项目
│   ├── hugo.toml                ← 配置（三语、每页24篇、RSS）
│   ├── content/zh-cn/blog/      ← 中文博客文章（Markdown）
│   │   ├── _index.md            ← 博客列表页标题/描述
│   │   └── posts/{slug}/index.md  ← 每篇文章一个文件夹
│   ├── layouts/blog/
│   │   ├── list.html            ← 卡片列表页模板（搜索+分页）
│   │   └── single.html          ← 文章页模板
│   ├── static/
│   └── deploy-blog.ps1          ← 一键构建+部署脚本
│
└── fengyuwang_5-deploy/         ← 线上部署目录
    ├── zh-cn/blog/               ← 中文博客（构建产物）
    ├── zh-hk/blog/               ← 繁体博客
    └── en/blog/                  ← 英文博客
```

## 当前文章结构

```yaml
---
title: "文章标题"
description: "从正文提炼的概要——显示在卡片列表上，正文中不出现"
slug: "url-slug"
draft: false
---

正文第一段（自然的文章开头，与 description 各自独立）

正文第二段...
```

### description 的定位

- **只用在一个地方**：卡片列表页的摘要是
- **不在文章页出现**
- 从正文中提炼/概括，而不是正文的一部分
- 它不是正文第一段，也不是正文的截取

### 正文第一段的定位

- 自然的文章开头，独立于 description
- 可以跟 description 意思相近，但写法不同
- 目标是让读者从第一句就被带入

## 模板逻辑

### 卡片列表页（list.html）

- 响应式网格：桌面多列（minmax(360px)），手机单列
- 卡片内容：标题 + 分隔线 + description（概要文字）+ 分类标签
- 整张卡片可点击直接进入文章
- 顶部搜索框：实时过滤标题/概要/标签
- 底部居中分页：← 上一页 + 页码按钮 + 下一页 → + GO 跳转
- 每页 24 篇

### 文章页（single.html）

- 顶部「← 返回博客」圆角按钮
- 标题
- 正文（从第一段开始自然阅读）
- 没有 description 显示，没有日期

## 文章改写/创作指南

### 核心流程（改写一篇文章的标准步骤）

1. **读全文**：理解核心观点、逻辑链条、情绪基调
2. **改写正文**：保持原有观点和案例，按风格要求重写
3. **提炼 description**：正文写完后，概括成一段话作为卡片摘要
   - 这条 description 不会出现在文章里，只用在卡片上
   - 它跟正文第一段可以有关联，但不是同一句话

### 风格要求

**松弛感、真诚叙事**——像随手发的朋友圈或小红书笔记，真实有细节。

**文风：**
- 冷静而戏谑，俏皮而深刻
- 短句为主，口语化但有质感
- 可以带小吐槽 + 小感悟（基于内容，不硬凹）
- 留白感强，不说满

**结构（每篇文章）：**
- 标题：带网感、有情绪或场景感
- 正文：真实体验 + 内心独白 + 细节场景
- 标签：3-5个，精准贴合内容 + 流量词

**禁忌：**
- ❌ "谁懂啊家人们" 等烂大街模板
- ❌ 刻意煽情、喊口号
- ❌ 第二人称对话（"我给你""你如何如何"）
- ❌ AI 对话感（"哈哈""我跟你说"）

**要求：**
- 不追求逻辑严密，真实就好
- 突出个人感受和生活洞察

## 工作流

### 写新文章

1. 在 `content/zh-cn/blog/posts/{slug}/` 下创建 `index.md`
2. 写 frontmatter：title、description、slug、draft
3. 正文从第一段开始自然书写
4. description 写完后提炼，跟正文各自独立
5. 运行 `deploy-blog.ps1` 构建并部署

### 翻译

复制 `zh-cn/blog/posts/{slug}/` 到 `zh-hk/` 和 `en/`，翻译 frontmatter 和正文。
