# 网站重新设计规划

> 基于对全部页面和博客文章的深度阅读，重新理解王丰羽的核心思想后制定的内容架构方案。
> 本计划仅涉及内容呈现结构调整，不触及 Hugo 主题文件和布局模板。

---

## 零、重新理解：你是谁

### 你的核心链条（从《第一性原理》提取）

```
赚钱 → 创造价值 → 解决问题 → 识别真需求 → 实践反馈 → 快速迭代 → 验证模型 → 系统复制 → 杠杆
```

这不是某一件事的方法论，这是你所有事的底层 DNA。

### 三条线，一个根

| 领域 | 表面理解 | 你的理解 |
|------|---------|---------|
| 市场学 | 推广/带货 | 数据洞察真需求 + 创意表达价值 = 双向联系 |
| 投资 | 炒股价/猜涨跌 | 理解企业价值 + 安全边际 + 长期复利 |
| 技术 | 写代码/搞技术 | 定义真问题 → 结构设计 → 工程交付 |

三项的共同底层：**都不是表面那层。** 每一项都往回追溯到"识别真需求"和"建立可复用的系统"。

### 你的独特位置：跨领域联结

**你同时懂三件事的联系——这才是真正的稀缺。**

纯市场人不懂数据架构 → 你懂。
纯程序员不懂商业判断 → 你懂。
纯投资者不懂获客逻辑 → 你懂。

从《「数据资产」的意识》就能看出：你一眼看穿商务表格乱的本质，是因为你**既懂商务BD在干什么，又懂工程化数据应该怎么管。** 这种跨界看清底层的能力，才是你的核心产品。

### 你的底色

**理性 · 客观 · 不坑人**——不是口号，是你每篇文章都在验证的东西。

## 一、现有网站诊断

### 对的（不需要动）

1. 多语言架构（zh-cn / en / zh-hk）—— 完整且正确
2. 三领域划分（市场 / 投资 / 技术）—— 本身就是你的核心框架
3. 能力结构页（展示所有技能）—— 信息密度够
4. 博客内容 —— 是你思想的完整输出
5. Hugo 主题和布局模板 —— 稳定、已交付

### 缺的（需要补）

1. **每页缺一个引领性的思想锚点** —— 现在每页从"市场学"标题直接进入内容，没有那个让人停一下的 punchline
2. **内容层级扁平** —— 所有内容平铺在卡片里，缺乏"大标题→子标题→正文"的递进节奏
3. **"三条线，一个根"的主题在页间没有贯穿** —— 每页是独立的岛，没有串联
4. **缺少跨页面的呼应路径** —— 比如在市场学页底部应该能自然连接到相关投资或技术内容

### 不需要动的

- Hugo 主题框架（layouts/blog/ 下的模板）
- hugo.toml 配置
- 博客文章的 markdown 文件本身
- 多语言路由结构

## 二、整体架构方案

### 核心原则

1. **每页一个有分量的思想锚点**（Apple "Switch to iPhone." 式的 punchline）
2. **每页内容按「大标题 → 介绍性用语 → 内容卡片」分层展开**
3. **页间通过三个领域的交叉引用形成闭环**
4. **所有改动仅限于各语言的 HTML 页面文件本身**（zh-cn/ 下的 index.html, mkt.html, invest.html, portfolio.html, capabilities.html）

### Apple 内容层级模式（参考）

```
Page Hero
  ├── 思想锚点（2-6字，撑起整页）
  │
  ├── Section 1：大标题（3-5字）
  │   └── 介绍性用语（一句话说明本节的要点）
  │       └── 内容卡片（现有内容的重新组织）
  │
  ├── Section 2：大标题
  │   └── 介绍性用语
  │       └── 内容卡片
  │
  └── 页尾：跨页链接 + 页脚
```

## 三、各页内容架构

### 3.1 首页（index.html）

**核心原则：首页是让人决定"这人值不值得深入了解"的地方。不是介绍页，不是服务页。**

页面结构：

```
思想锚点 + 一句话说明你是谁
  └── 三条入口（三个领域，各自带一句话引子）
      └── 最新思考（博客 + 动态）
          └── 联系
```

---

**思想锚点**：`理性 · 客观 · 不坑人`

**副线**（可选，一句话说明你是谁）：*市场策略 · 价值投资 · 软件工程*

**CTA 逻辑**：不是"探索服务"，不是"联系我"——而是让真被打动的人有一个自然的下步动作。

候选项（需要决定）：
- "开始阅读" / "阅读文章" → 引导到博客，适合想先了解你思考的人
- "开始对话" → 引导到邮箱/联系，适合已经确定想聊的人
- 也可以不加显式 CTA，让三条 track card 本身作为入口

### 3.5 能力结构（capabilities.html）

**核心原则：这张页不能是"技能说明书"，要让人扫一眼就知道你能帮他什么。**

现有技能树（熟练掌握/略懂/未来要学/谢绝）是内容基础设施，但不是主展示。主展示是 punchline 驱动的"能力叙事"。

---

**思想锚点**：`三条线，一个根`

你有三个方向的能力，但它们不是孤立的——从同一个底层逻辑延伸出来。

页首配一句话通俗解释，比如：*我做的事跨三个领域，但底层的思考方式只有一套。*

---

| Section | 大标题 | Punchline（扫一眼就懂） | 内容说明 |
|---------|--------|------------------------|---------|
| 1 | 市场理解力 | 让人想买，而不是让人烦 | 品牌策略、消费者洞察、CRM、AI 营销、活动策划——把产品翻译成消费者愿意听的语言 |
| 2 | 技术实现力 | 想得到，做得出 | 从 Flutter 到 Python 到 Docker，能把想法变成能跑、能交付、能维护的产品 |
| 3 | 商业判断力 | 知道什么值得做 | 价值投资、财务分析、概率思维——在不确定中判断什么方向是对的 |
| 4 | 三角闭环 | 一个打三个 | 三个能力不是并列，是互相喂的。市场发现机会，技术实现产品，商业判断方向——来回验证 |
| 5 | 完整清单 | — | 现有技能树（熟练掌握/略懂/未来要学/谢绝），作为可展开的参考附录 |

**Punchline 的作用**：即使不看下面具体内容，光扫这五行 punchline，任何人都能立刻理解「这个人能干什么、和别人的能力有什么不同」。

结构调整要点：
- **重点变化：技能树从"主展示"降级为"附录"**。主展示是 4 个 punchline 驱动的能力区块 + 1 个三角闭环说明
- "熟练掌握 / 略懂 / 未来要学 / 谢绝" 的完整清单保留——作为可以展开/收缩的详细参考，不删除
- "谢绝"部分保留位置，是你声音的一部分
- 第 4 节「三角闭环」是整个能力页的核心差异点——大多数人的能力是平行的，你的能力是互相增强的。这是整页唯一新写的内容，需要 2-3 句话说清楚三个能力怎么互相喂

## 五、CTA 策略（全站通用）

### 核心原则

**CTA 文案不应该是功能描述（"探索服务""联系我"），而应该是"一个真被我打动的人，最自然会点下去的那个东西"。**

### 每页 CTA 逻辑

| 页面 | 用户看完的感受 | 他最自然会想做的事 | 建议 CTA |
|------|---------------|-------------------|---------|
| 首页 | "这人有点东西" | 想看他到底写过什么 | 引导到博客 或 三条 track card 本身作为入口（不加 CTA） |
| 市场学 | "市场还能这么想" | 想看案例/联系聊项目 | 案例展示区域 → 底部联系 |

| 技术 | "是个会做事的" | 想看代码/聊技术 | GitHub / LinkedIn / 联系 |
| 能力结构 | "这人能做我需要的" | 想直接联系 | 联系 |

- 当前页面的 hover 效果
- 深色模式切换
